# REQ-003: 效能與本地端資料庫優化原則 (Performance & DB Optimization)

## Description
為了維持跨平台桌面端應用 (Tauri) 的順暢度，特別是在處理大量表格 (如 `ResultTable`) 與頻繁更新的圖表時，必須嚴格遵守以下效能優化準則：

### 1. React 渲染優化 (避免渲染瀑布)
- **穩定 Key**: 列表渲染時禁止使用單純的 array `index` 作為 key，必須使用具體識別字串（如 `${row.stock_id}-${index}`）避免不必要的節點重建。
- **組件記憶化 (Memoization)**: 表格列 (`ResultTableRow`) 與圖表組件 (如 `RowChart`, `DailyUltraTinyLineChart` 等) 必須使用 `React.memo` 包裝，防止全域狀態 (如 `chartType`) 變更時波及全部子組件。
- **穩定化依賴 (Dependencies)**: 傳遞給子組件的 `columns`、複雜物件設定或回呼函數，必須妥善使用 `useMemo` 與 `useCallback` 包覆。

### 2. SQLite 查詢與快取優化
- **查詢去重與快取**: 在與 SQLite 溝通時，必須透過全域快取機制 (`useQueryCache.ts`) 阻擋重複查詢。相同的查詢在短時間 (如 30 秒) 內不可向資料庫重複發送。
- **統一介面**: 所有圖表拉取資料都應透過統一的 Hook (如 `useChartData.ts`) 管理請求生命週期。
- **穩定化查詢函數**: Hook (例如 `useFindStocksByPrompt`) 返回的查詢函式必須套用 `useCallback` 穩定參考，避免 `useEffect` 的依賴陣列無限重新觸發。
- **性能監控**: 開發時必須留意 `DatabasePerformanceMonitor` 避免 N+1 查詢發生。
