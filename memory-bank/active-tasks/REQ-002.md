# REQ-002: 策略資料結構與特殊規則約束 (StorePrompt / PromptItem)

## Description
為了確保系統內自訂選股策略的結構一致性，所有的策略建構、讀取與儲存，均必須遵守以下 TypeScript 型別定義與邏輯：

1. **`StorePrompt` (單一條件規則)**:
   包含兩個時間點及對應的指標與比較運算子：
   - `day1`: string (e.g., "本週", "今天")
   - `indicator1`: string (e.g., "收盤價", "ma5")
   - `operator`: string (e.g., "大於", "小於")
   - `day2`: string (e.g., "上週", "自定義數值")
   - `indicator2`: string (e.g., "收盤價", 或者當 day2 為自定義數值時填寫數字字串)

2. **`Prompts`**:
   `StorePrompt[]` 的陣列。

3. **`PromptValue` (跨時框條件集合)**:
   必須包含 `daily`, `weekly`, `hourly` 三種時框，值為 `Prompts`：
   ```typescript
   export type PromptValue = {
     daily: Prompts;
     weekly: Prompts;
     hourly: Prompts;
   };
   ```

4. **`PromptItem` (完整策略物件)**:
   封裝了策略名稱與條件集合，並帶有索引用：
   ```typescript
   export type PromptItem = {
     name: string;
     conditions: PromptValue;
     index: number;
   };
   ```

任何涉及策略編輯器、自動產生策略、資料驗證或前後端策略交換的功能，皆「必須」以這些型別作為絕對標準。
